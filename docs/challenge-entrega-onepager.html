Entrega Challenge Sr Fullstack (Microservicios)

Objetivo: evolucionar el backend NestJS + PostgreSQL a un enfoque event-driven, resolver problemas estructurales mínimos y demostrar el flujo completo en una web React.

Resultado: backend y frontend funcionando end-to-end con permisos por rol, lifecycle de producto (crear, editar, activar, desactivar), eventos de dominio persistidos en product_event y timeline visible en UI.

Qué resolví

  • Agregué /health y /api/health para integración y monitoreo.
  • Habilité CORS para frontend/backend separados.
  • Completé endpoints de lectura: /product, /category, /user, /role.
  • Completé lifecycle con /product/:id/deactivate.
  • Definí rol único efectivo para usuarios no-admin con /role/change.

Event-Driven implementado

  • Eventos: product.created, product.activated, product.deactivated.
  • Listener 1: registra PRODUCT_CREATED/ACTIVATED/DEACTIVATED en product_event.
  • Listener 2: al activar, setea activatedAt en producto.
  • Proyección con índices por productId, type y occurredAt.
create -> emit event -> listener -> product_event -> timeline UI activate/deactivate -> emit event -> listener -> timeline UI

Flujo demo (6 pasos)

  1. Admin inicia sesión y cambia un usuario a Merchant.
  2. Merchant crea producto con datos completos.
  3. Merchant edita detalles del producto.
  4. Merchant activa producto.
  5. Merchant desactiva producto.
  6. Se valida timeline con 3 eventos de dominio.

  • Auth: /auth/register, /auth/login
  • Roles: /role, /role/change (/role/assign deshabilitado)

  • Categorías: /category
  • Productos: /product, /product/:id, /product/:id/events
  • Lifecycle: /create, /details, /activate, /deactivate
